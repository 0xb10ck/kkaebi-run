## 0. MVP 범위 (Phase 1 Scope)

본 문서는 GDD v2.0의 "Phase 1 — 프로토타입(MVP)" 항목을 구체 수치로 확정한다.

| 항목 | 범위 |
|------|------|
| 플레이 가능 캐릭터 | **뚝딱이** 1종 |
| 무대 | **1장 두멍마을 — 1스테이지(5분)** 1종 |
| 일반 몬스터 | **도깨비불 / 달걀귀신 / 물귀신** 3종 |
| 미니보스 / 챕터 보스 | **없음** (Phase 2) |
| 스킬 | **오행 5종 각 Lv1** (도깨비불 / 서리고리 / 덩굴채찍 / 금빛방패 / 바위투척) |
| 레벨업 | 레벨업 시 **3택 선택지** |
| 메타 성장 / 도감 / 상점 | **없음** (Phase 2 이후) |
| 사운드 | 효과음 최소 셋, BGM 1곡 (사운드는 본 문서 범위 외) |
| 빌드 타겟 | **HTML5 웹** |

MVP 클리어 조건: **5분 생존** 또는 **HP 0으로 사망**. 어느 쪽이든 결과 화면 → 재시작 흐름으로 연결한다.

## 1. 플레이어 — 뚝딱이 스탯

### 1.1 기본 스탯

| 항목 | 값 | 비고 |
|------|----|------|
| **시작 HP** | `100` | 정수 |
| **최대 HP** | `100` (MVP에서는 상승 없음) | 회복 시 캡 |
| **이동 속도** | `180 px/s` | 가상 픽셀 기준, 디아고날 정규화 |
| **히트박스 반지름** | `12 px` | 원형 |
| **무적 시간 (피격 후)** | `0.6 s` | 피격 직후 모든 접촉/장판 데미지 무시 |
| **HP 자연 회복** | 없음 | MVP 단계 |

### 1.2 기본 무기 — 도깨비 방망이 (회전 타격)

뚝딱이는 **자동 공격**으로 자기 주위를 회전하는 도깨비 방망이를 휘두른다. 별도 입력 불필요.

| 항목 | 값 | 비고 |
|------|----|------|
| **회전 반경** | `80 px` | 캐릭터 중심에서 방망이 헤드까지 |
| **회전 속도** | `360 °/s` | 1초에 1회전 |
| **방망이 헤드 히트박스** | 캡슐 형태 폭 `28 px`, 길이 `40 px` | 적과 겹치는 동안 데미지 판정 |
| **데미지 (히트당)** | `12` | 정수, 크리티컬 미적용 |
| **같은 적 재타격 쿨다운** | `0.4 s` | 한 번 맞은 적은 0.4초 동안 다시 맞지 않음 |
| **회전 방향** | 시계 방향 | 시각적 일관성 |
| **공격 활성** | 게임 시작과 동시에 항상 ON | 정지 시에도 회전 유지 |

> **밸런스 메모**: 이 기본 무기만으로 도깨비불(HP 8)은 1히트, 달걀귀신(HP 15)은 2히트, 물귀신(HP 40)은 4히트에 처치된다.

| **드롭 보석 가치 (EXP)** | `1` |
| **드롭 확률** | 100% (반드시 1개) |
| **금화 드롭** | 10% 확률로 `1냥` |

| **스프라이트 크기** | `24×24` |

| 항목 | 값 |
|------|----|
| **HP** | `15` |

| **접촉 데미지** | `6` |
| **드롭 보석 가치 (EXP)** | `2` |
| **드롭 확률** | 100% |
| **금화 드롭** | 15% 확률로 `1냥` |

### 2.3 물귀신 (탱키 / 둔화 유발)

| 항목 | 값 |
|------|----|
| **HP** | `40` |
| **이동 속도** | `50 px/s` |
| **접촉 데미지** | `10` |
| **접촉 시 추가 효과** | 플레이어 **이동 속도 -30%**, 지속 `1.0 s` (재접촉 시 갱신) |
| **드롭 보석 가치 (EXP)** | `5` |
| **드롭 확률** | 100% |
| **금화 드롭** | 50% 확률로 `2냥` |
| **히트박스 반지름** | `20 px` |
| **스프라이트 크기** | `48×48` |

### 2.4 시간 경과 강화 곡선

게임 진행 시간 `t`(초)에 따라 모든 몬스터에 다음 보정이 적용된다. 강화는 **스폰 시점**에 결정되며, 이미 필드에 있는 개체는 변하지 않는다.

```
HP 배율    = 1.00 + floor(t / 30) * 0.10     # 30초마다 +10%
속도 배율  = 1.00 + floor(t / 60) * 0.05     # 60초마다 +5%
데미지 배율= 1.00 + floor(t / 90) * 0.10     # 90초마다 +10%
EXP 배율   = 1.00 (고정)                       # 보상은 일정
```

**5분(t=300) 시점 누적 보정**:
- HP × `2.00` (10단계 누적)
- 속도 × `1.25` (5단계 누적)
- 데미지 × `1.30` (3단계 누적)

> **예시 — 5분차 도깨비불**: HP `16`, 속도 `100 px/s`, 접촉 데미지 `5.2 → 5`(내림). 처치 EXP는 그대로 `1`.

## 3. 경험치 곡선 및 레벨업 3택

### 3.1 EXP 보석 수집

- 몬스터 처치 시 위치에 EXP 보석이 드롭된다.
- **자동 수집 반경**: `60 px` (보석이 플레이어 쪽으로 빨려옴, 흡인 속도 `300 px/s`).
- **자석 효과 보석**: 화면 내 모든 보석을 즉시 수집(현재 MVP에서는 미구현, Phase 2).

### 3.2 레벨업 요구 EXP 곡선

```
Lv N → Lv (N+1) 에 필요한 누적 보석 EXP:

  required(1)  =  5
  required(2)  =  8
  required(3)  = 12
  required(4)  = 16
  required(N>=5) = 16 + (N - 4) * 6
```

| 도달 레벨 | N→N+1 필요 EXP | 누적 EXP |
|-----------|----------------|----------|
| 1 → 2 | 5 | 5 |
| 2 → 3 | 8 | 13 |
| 3 → 4 | 12 | 25 |
| 4 → 5 | 16 | 41 |
| 5 → 6 | 22 | 63 |
| 6 → 7 | 28 | 91 |
| 10 → 11 | 52 | 281 |
| 15 → 16 | 82 | 616 |

> **밸런스 메모**: 5분 동안 평균 600~900 EXP 획득을 가정하면, 클리어 시 도달 레벨은 약 **Lv 14~17**.

### 3.3 레벨업 3택 규칙

레벨업 발생 시 게임은 **일시 정지**(time scale = 0)되고 카드 3장이 표시된다.

**카드 풀 구성 규칙**:
1. **소지 스킬 우선**: 이미 보유한 스킬 중 **Lv < 5(MVP에서는 Lv 1만 존재하므로 사실상 미사용)** 인 것은 "+1 레벨" 카드로 등장 가능. (MVP에서는 모든 스킬 Lv 1 단일 → 이미 보유한 스킬은 풀에서 제외)
2. **미보유 스킬**: 5종 중 아직 획득하지 않은 스킬은 "신규 획득" 카드로 등장.
3. **풀 부족 시 폴백**: 남은 미보유 스킬이 3개 미만이면 부족분은 다음 보너스 카드로 채운다.
   - **`최대 HP +20`** (즉시 +20, 현재 HP도 +20 회복)
   - **`이동 속도 +5%`** (영구, 누적 가능)
   - **`경험치 획득량 +10%`** (영구, 누적 가능)
4. 같은 화면에 **동일 카드는 한 번만** 등장한다.

**선택 UX**:
- 카드 3장을 화면 중앙에 가로 배치.
- 마우스 호버 시 카드 1.05배 확대.
- 클릭으로 선택, 또는 키보드 `1` / `2` / `3` 단축키.
- 선택 즉시 게임 재개.

**5종 모두 획득 후 레벨업**:
- 보너스 카드 3장(위 3종 중 무작위, 중복 허용 안 됨 — 부족하면 같은 항목이라도 노출)으로 채운다.

## 5. 1스테이지(5분) 스폰 패턴 및 난이도 곡선

### 5.1 공통 규칙

- **스테이지 길이**: `300 s` (5분). 종료 시 자동 클리어 결과 화면.
- **스폰 위치**: 카메라 시야 바깥 가장자리에서 무작위(상/하/좌/우 면당 25% 확률). 시야 바깥 `60~120 px` 지점.
- **동시 존재 상한**: 화면 내 일반 몬스터 동시 최대 `120`체. 초과 시 가장 멀리 있는 개체를 디스폰.
- **무리 스폰(달걀귀신)**: 4마리가 가로 30 px 간격으로 묶여 같은 방향에서 진입.
- **스폰 단위**: "한 번에 N마리"는 별도 명시가 없으면 단일 개체.

### 5.2 시간대별 스폰 곡선

| 구간 (mm:ss) | 도깨비불 | 달걀귀신 (4마리 무리 단위) | 물귀신 | 비고 |
|--------------|----------|--------------------------|--------|------|
| **0:00 – 0:30** | 8마리 / 15s | — | — | 튜토리얼 구간, 사방에서 산발 진입 |
| **0:30 – 1:30** | 12마리 / 15s | 1무리 / 20s (4마리) | — | 달걀귀신 무리 출현 시작 |
| **1:30 – 2:30** | 16마리 / 15s | 2무리 / 20s (8마리) | 1마리 / 30s | 첫 물귀신 등장 |
| **2:30 – 3:30** | 24마리 / 15s | 3무리 / 20s (12마리) | 1마리 / 20s | 압박 본격화 |
| **3:30 – 4:30** | 32마리 / 15s | 4무리 / 20s (16마리) | 2마리 / 20s | 양쪽 가장자리에서 동시 스폰 |
| **4:30 – 5:00** | 40마리 / 10s | 6무리 / 15s (24마리) | 3마리 / 15s | **최종 총공세** |

> **세부 스폰 룰**: 각 "N마리 / 15s"는 15초 윈도우 동안 균등 분산(예: 8마리/15s → 1.875초 간격). 무리 스폰은 윈도우 시작 시점에 묶음 단위로 한 번에 진입.

### 5.3 난이도 검증 기준

밸런스 검증을 위해 다음 체크포인트를 목표로 한다(QA 단계에서 측정):

| 시점 | 기대 도달 레벨 | 기대 처치 수 (누적) | 기대 HP 잔량 |
|------|---------------|-------------------|--------------|
| 1:00 | Lv 3~4 | 30~50 | 80~100 |
| 2:30 | Lv 7~9 | 150~220 | 50~90 |
| 4:00 | Lv 11~14 | 400~600 | 30~80 |
| 5:00 | Lv 14~17 | 700~1000 | 1~80 |

> **밸런스 목표**: 5분 클리어 성공률 약 **30~50%** (스킬 픽 운에 따라). 첫 30초 안에 사망하는 일이 없어야 한다.

### 5.4 디버프 효과 누적 규칙

- 물귀신 둔화(-30% 이속, 1초)와 서리고리에 의한 몬스터 둔화(-25%, 1.5초)는 **서로 다른 주체에 적용**되므로 충돌 없음.
- 동일 효과 중복 적용 시 **지속 시간만 갱신**, 효과 강도는 합산하지 않는다.

## 6. HUD 표시 요소 및 위치

기준 해상도 **1280×720**. 좌표는 화면 좌상단 (0,0) 기준 픽셀.

```
┌──────────────────────────────────────────────────────────┐
│  [HP 320×32]              [⏱ 03:24]            [🪙 12냥] │
│  Lv. 7                                                   │
│                                                          │
│                                                          │
│                                                          │
│                          (게임 필드)                      │
│                                                          │
│                                                          │
│                                          [화][수][목][금][토] │
│ ════════════════ EXP 1280×16 ═══════════════════════════ │
└──────────────────────────────────────────────────────────┘
```

| 요소 | 위치 (x, y) | 크기 | 정렬 | 비고 |
|------|-------------|------|------|------|
| **HP 바** | (20, 20) | 320×32 | 좌상단 | 호랑이 가죽 프레임 + 붉은 게이지. 현재 HP / 최대 HP 텍스트 중앙 오버레이 (`78 / 100` 형식) |
| **레벨 표시** | (20, 60) | 폭 자동, 높이 20 | 좌상단 | `Lv. N` 형식, 흰색 외곽선, 검정 본체 |
| **타이머** | (640, 24) | 자동 | 상단 중앙 (앵커 중앙) | 형식 `MM:SS`, 붓글씨 폰트 32 px, 시간이 남은 게 아니라 **경과 시간**을 표시 |
| **금화** | (1260, 20) | 자동 | 우상단 (우측 정렬) | 엽전 아이콘 24×24 + 숫자 + `냥`. 예: `🪙 12냥` |
| **경험치 바** | (0, 704) | 1280×16 | 화면 하단 풀 와이드 | 신목 금빛(#FFD700) 게이지, 빈 부분 어두운 갈색. 우측 끝에 `Lv N → N+1` 텍스트 |
| **스킬 아이콘 슬롯** | 우하단, (1280 − 20 − (5−i)×48, 660) `i = 0..4` | 각 40×40, 간격 8px | 우하단, 우→좌로 채워짐 | 보유 스킬만 표시. 빈 슬롯은 회색 외곽선. 액티브 스킬은 쿨다운 시계 효과 |

**색상 토큰** (`docs/kkaebi-run-asset-concept.md` 컬러 팔레트 기반):
- HP 바 게이지: `#E03C3C`, 외곽선 `#0F0F15`
- 경험치 바 게이지: `#FFD700`, 트랙 `#45455A`
- 타이머/레벨 텍스트: 본체 `#F0EDE6`, 외곽선 `#2A2A35`
- 금화 텍스트: `#E0C23C`

**가시성 규칙**:
- 모든 HUD 요소는 게임 필드 위에 그려진다(z-order 최상위).
- 레벨업 3택 모달이 떠 있는 동안 HUD는 그대로 표시되지만 게임 필드는 어둡게(alpha 0.5) 처리한다.

## 7. 사망 → 결과 화면 → 재시작 흐름

### 7.1 흐름 다이어그램

```
HP <= 0 도달
   │
   ▼
[연출 1.0s]
  · time scale 0.3 (슬로모션)
  · 화면 어둡게 페이드 (alpha 0 → 0.6)
  · 도깨비 연기 사라짐 이펙트
  · "퇴장" 효과음 1회
   │
   ▼
[결과 화면 페이드인 0.5s]
  · 통계 카운트업 애니메이션 (각 수치 0 → 실제값, 0.8s 동안)
   │
   ├──[클릭: 다시 도전하기]──▶ 스테이지 즉시 재시작 (모든 상태 초기화)
   │
   └──[클릭: 메인 메뉴]    ──▶ 메인 화면으로 전환 (MVP에서는 메인 화면 = 시작 버튼만 있는 단순 화면)
```

5분 생존 클리어 시에도 동일한 결과 화면 흐름을 사용하되, 제목/부제 문구만 다르다(§7.3).

### 7.2 결과 화면 — 사망 시 한국어 문구

```
┌───────────────────────────────────────────┐
│                                           │
│           도깨비가 잠들었습니다             │  ← 제목 (붓글씨 폰트, 48px)
│                                           │
│   신목의 기운을 다시 모으기까지            │  ← 부제 (24px)
│      잠시 기다려 주십시오.                 │
│                                           │
│   ───────────  결 산  ───────────         │  ← 구분선
│                                           │
│        생존 시간       02분 47초          │
│        처치한 잡귀       312마리          │
│        도달 레벨        Lv. 9             │
│        획득한 금화       18냥             │
│                                           │
│                                           │
│      ┌─────────────────┐                  │
│      │   다시 도전하기   │  ← 강조 버튼 (빨강) │
│      └─────────────────┘                  │
│      ┌─────────────────┐                  │
│      │ 메인 메뉴로 돌아가기 │ ← 보조 버튼 (회색) │
│      └─────────────────┘                  │
│                                           │
└───────────────────────────────────────────┘
```

**문자열 명세 (사망 시)**:

| 위치 | 문자열 (한국어 존댓말) | 폰트/스타일 |
|------|------------------------|--------------|
| 제목 | `도깨비가 잠들었습니다` | 붓글씨, 48px, 검정 본체 + 흰 외곽선 |
| 부제 | `신목의 기운을 다시 모으기까지 잠시 기다려 주십시오.` | 본문, 24px |
| 결산 헤더 | `───── 결 산 ─────` | 본문, 20px, 옅은 회색 |
| 통계 라벨 1 | `생존 시간` / 값: `MM분 SS초` | 본문, 22px |
| 통계 라벨 2 | `처치한 잡귀` / 값: `N마리` | 본문, 22px |
| 통계 라벨 3 | `도달 레벨` / 값: `Lv. N` | 본문, 22px |
| 통계 라벨 4 | `획득한 금화` / 값: `N냥` | 본문, 22px |
| 강조 버튼 | `다시 도전하기` | 28px, 흰색, 빨강 배경(`#E03C3C`) |
| 보조 버튼 | `메인 메뉴로 돌아가기` | 24px, 흰색, 회색 배경(`#45455A`) |

### 7.3 결과 화면 — 5분 클리어 시 한국어 문구

같은 레이아웃을 사용하되 제목/부제만 교체한다.

| 위치 | 문자열 |
|------|--------|
| 제목 | `스테이지를 완수하셨습니다` |
| 부제 | `두멍마을의 밤이 다시 조용해졌습니다. 도깨비님 덕분입니다.` |

나머지 통계, 버튼 문구는 동일.

### 7.4 재시작 동작 규칙

- **`다시 도전하기`** 클릭 시:
  - 현재 씬을 완전히 리로드(`get_tree().reload_current_scene()` 등).
  - 모든 상태(HP, 레벨, EXP, 보유 스킬, 금화, 스폰 타이머, 시간 경과 강화)는 초기값으로 리셋.
  - 페이드 인/아웃 0.4s.

- **`메인 메뉴로 돌아가기`** 클릭 시:
  - 메인 메뉴 씬으로 전환.
  - MVP의 메인 메뉴는 게임 로고 + `시작하기` 버튼 한 개만 표시한다(메타 강화 화면 없음).

### 7.5 기타 사용자 노출 문구 (한국어 존댓말)

스테이지 진행 중 다음 시점에 토스트 메시지가 표시된다. 모두 화면 상단 중앙, 1.5초 페이드 인/아웃, 폰트 28px.

| 트리거 | 문구 |
|--------|------|
| 스테이지 시작 직후 (0:00) | `도깨비님, 두멍마을을 지켜 주십시오.` |
| 첫 레벨업 (Lv 1 → 2) | `힘이 차오릅니다. 신통을 골라 주십시오.` |
| 첫 물귀신 등장 (1:30) | `물귀신이 다가옵니다. 발을 조심하십시오.` |
| 4:00 도달 | `검은 안개가 짙어집니다. 곧 끝이 보입니다.` |
| 4:50 도달 (마지막 10초) | `잠시만 더 버텨 주십시오.` |
| 5:00 클리어 직전 | `두멍마을의 새벽이 밝아옵니다.` |

> **주의**: 모든 사용자 노출 문구(메뉴, 버튼, 토스트, 결산, 토큰)는 **한국어 존댓말**로 작성하며, 해라/해체/반말은 금지한다. 영어 단어 단독 노출도 금지한다(예: `Game Over` ❌ → `도깨비가 잠들었습니다` ✅).

## 9. MVP 완료 정의 (Definition of Done)

다음 조건을 **모두** 만족하면 Phase 1 MVP가 완료된 것으로 본다.

1. 뚝딱이 1종으로 1280×720 웹 빌드에서 5분 스테이지 플레이가 가능하다.
2. 도깨비불·달걀귀신·물귀신 3종이 §5의 곡선에 따라 스폰되고, 시간 경과 강화가 적용된다.
3. 오행 스킬 5종이 §4 수치대로 동작한다(획득·발동·쿨다운·데미지).
4. 레벨업 시 3택 모달이 뜨고 게임이 일시 정지되며, 카드 선택으로 스킬이 즉시 활성화된다.
5. HUD 5요소(HP, 레벨, 타이머, 금화, 경험치, 스킬 아이콘)가 §6 위치에 표시된다.
6. HP 0 또는 5분 도달 시 §7의 결과 화면이 뜨고, `다시 도전하기` / `메인 메뉴` 버튼이 모두 동작한다.
7. 모든 사용자 노출 문구가 한국어 존댓말로 표시되며, 깨진 문자가 없다.
8. Godot 4.x 웹 빌드(`html`/`js`/`wasm`/`pck`)가 정상 생성되고 브라우저에서 60fps 근처(>= 45fps)로 구동된다.

## 10. Phase 2 이후로 미루는 항목 (Out of Scope)

명시적으로 **MVP에서는 만들지 않는다**:

- 스킬 레벨업(현재 모든 스킬 Lv1 단일), 진화/시너지 스킬
- 미니보스, 챕터 보스, 보스 페이즈
- 메타 재화(도깨비 구슬, 신목 이파리), 영구 강화
- 신목 진행, 캐릭터 해금(뚝딱이 외 5종)
- 도감, 도전 과제, 일일/주간 도전
- 사운드(BGM, 풀 SE 셋) — 최소 효과음 외
- 모바일 빌드, 게임패드/터치 입력
- 환경 인터랙션(독 늪, 부적 기둥, 신령 제단 등)
- 랜덤 이벤트(도깨비 시장, 신령의 축복 등)
- 저주 선택지, 8픽 슬롯 확장
