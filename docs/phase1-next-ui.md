# 깨비런 Phase 1 — 다음 단계 UI 와이어프레임

> **작성일**: 2026-05-13
> **연관 스펙**: `docs/phase1-next-spec.md`
> **대상 해상도**: 480 × 854 (세로 / portrait)
> **목적**: HUD, 사망/결과 화면, 일시정지 메뉴의 레이아웃·문구·상태 전환을 엔지니어가 그대로 구현할 수 있는 수준으로 명세

이 문서는 시각 디자인(폰트 가족, 그림자, 픽토그램 디테일)을 다루지 않습니다. 위치·크기·색상·문구·상태 전환만 다룹니다.

---

## 0. 공통 — 색상 팔레트 (오방색)

| 토큰 | HEX | 용도 |
|------|-----|------|
| `color.red` | `#E03C3C` | HP 바 채움, 임박 시간 경고 |
| `color.blue` | `#3C7CE0` | 정보 강조, 보조 텍스트 |
| `color.green` | `#4CAF50` | 회복/긍정 (이번 단계 미사용, 예약) |
| `color.paper` | `#F0EDE6` | 본문 텍스트 (한지 흰색) |
| `color.gold` | `#E0C23C` | EXP 바, 주 액션 버튼, 금화 |
| `color.overlay` | `#00000080` | 바·패널 어두운 배경 (alpha 50%) |
| `color.panel` | `#1A1612` | 결과/일시정지 패널 본체 |
| `color.disabled` | `#7A7670` | 비활성 버튼 텍스트 |

> 모든 사용자 노출 텍스트는 **한국어 존댓말**. 시스템 라벨(`레벨`, `금화`, `남은 시간`)은 평서 명사형 허용.

### 1.3 동작 규칙

| 트리거 | 결과 |
|--------|------|
| `Player.hp_changed(hp, max_hp)` | HP 바 채움 비율과 텍스트 갱신 (즉시) |
| `Player.exp_changed(exp, exp_to_next)` | EXP 바 채움 비율 갱신 (즉시) |
| `Player.level_up(level)` | `Lv. N` 텍스트 갱신, 0.2초간 1.0→1.2→1.0 스케일 트윈 |
| `Player.gold_changed(amount)` | `금화 N` 라벨 갱신 (Phase 1 안전망: signal이 없으면 `0` 고정) |
| `Main.time_left` 매 프레임 | 타이머 텍스트 갱신, 남은 시간 ≤ 10초이면 `color.red` 로 색 전환 |
| ⏸ 버튼 클릭 또는 `ESC` 키 | `Paused` 상태로 전환 (§3 참조) |

### 1.4 한국어 문구

| 라벨 | 문구 | 비고 |
|------|------|------|
| 타이머 | 숫자만 표시 (`MM:SS`) | 라벨 텍스트 없음 — 시인성 우선 |
| 레벨 | `Lv. {N}` | `N`은 정수, 1부터 |
| 금화 | `금화 {N}` | `N`은 정수, 0부터 |
| HP | `{cur} / {max}` | 숫자만 |

### 1.5 상태 (HUD 자체)

| 상태 | 표시 |
|------|------|
| 정상 | 위 1.2 그대로 |
| 마지막 10초 | 타이머 텍스트 색 `color.red`, 0.5초 간격 깜빡임(alpha 1.0↔0.6) |
| 일시정지 중 | 모든 요소 그대로 유지, 단 ⏸ 버튼 글리프는 ▶로 변경 |
| 결과 화면 표시 중 | HUD는 그대로 보이되 입력은 무시 (결과 패널이 위에 깔림) |

### 2.4 동작 규칙

| 트리거 | 결과 |
|--------|------|
| `Player.died` emit | `Playing → Dead`. 결과 화면 **사망 모드**로 노출 |
| `Main.time_left ≤ 0` | `Playing → Cleared`. 결과 화면 **클리어 모드**로 노출 |
| 결과 화면 진입 | `get_tree().paused = true`, 0.25초 페이드인 트윈 |
| `다시 시작` 클릭 / 엔터 / 스페이스 | `get_tree().paused = false`, `reload_current_scene()` |
| `메인 메뉴로` 클릭 | **반응 없음** (`disabled=true`. focus 가능하지만 활성 안 됨) |

```
       │                    │                      │
       │             ┌─────────────┐               │
       │             │   Cleared   │               │
       │             │ (결과·클리어)│               │
       │             └──────┬──────┘               │
       │                    │                      │
       │   다시 시작        │  다시 시작            │
       │   (paused=false,   │  (paused=false,      │
       │    reload_scene)   │   reload_scene)      │
       └────────────────────┴──────────────────────┘
                            ▼
                       (Playing 재시작)

       ※ `메인 메뉴로` 전이는 Phase 1 비활성 — 본 다이어그램에서도 점선 처리
```

### 4.2 전이 표

| From | Event | To | 사이드 이펙트 |
|------|-------|----|--------------|
| `Playing` | `ESC` 키 / ⏸ 버튼 | `Paused` | `get_tree().paused = true`, 일시정지 메뉴 표시 |
| `Paused` | `ESC` / `계속` 클릭 | `Playing` | `get_tree().paused = false`, 일시정지 메뉴 제거 |
| `Paused` | `다시 시작` 클릭 | `Playing` (새 판) | `paused = false`, `reload_current_scene()` |
| `Playing` | `Player.died` emit | `Dead` | `paused = true`, 결과 화면 사망 모드 |
| `Playing` | `time_left ≤ 0` | `Cleared` | `paused = true`, 결과 화면 클리어 모드 |
| `Dead` / `Cleared` | `다시 시작` 클릭 / Enter | `Playing` (새 판) | `paused = false`, `reload_current_scene()` |
| `Dead` / `Cleared` | `메인 메뉴로` | (비활성) | — |
| `Paused` | `Player.died` | (불가) | `Player`도 `PROCESS_MODE_PAUSABLE`이므로 발생 불가 |

### 4.3 우선순위 규칙

1. **결과 화면 > 일시정지 메뉴**
   - 결과 화면 표시 중에는 ESC / ⏸ 입력 무시.
2. **`Player.died`와 `time_left ≤ 0` 동시 발생**
   - `Player.died`를 우선. 죽은 채로 클리어 처리되지 않도록 `_process` 도입부에서 `Main.is_dead` 체크.
3. **씬 리로드 직후 첫 프레임**
   - `Main._ready()`에서 `get_tree().paused = false`를 명시적으로 호출하여, 이전 결과 화면의 pause 상태가 누수되지 않도록 한다.

## 6. 핸드오프 노트 (Engineer / QA)

### 6.1 Engineer

- **HUD** 는 Phase 1 스펙(`docs/phase1-next-spec.md` §3.1)의 좌표·크기 대신 **본 문서 §1.2의 값**을 사용. (HP 바 280×14, 우상단 타이머, ⏸ 버튼 추가)
- 본 단계 구현 우선순위:
  1. HUD (HP / EXP / 레벨 / 타이머)
  2. 결과 화면 (사망·클리어 모드)
  3. 일시정지 메뉴
  4. ⏸ 버튼 + `pause` 입력맵
  5. `금화 N` 표시 (signal 부재 시 `0` 고정 가능)
- **상태 머신은 Main 씬에 enum + match 한 군데에만 둔다.** 결과 화면이 자체적으로 paused를 만지면 일시정지와 경합한다.
- 결과 화면의 `획득 금화` 통계는 Phase 1에서 `0` 표시여도 통과. 단 레이아웃 자리는 비우지 않는다 (4행 고정).
- 일시정지 메뉴와 결과 화면은 **서로 다른 CanvasLayer**로 두고 레이어 z-index로 결과 화면을 위에 둔다. 동시 출현 가능성을 코드 분기 없이 시각 우선순위로만 해결.

### 6.2 QA — UX 검증 체크리스트

- [ ] HUD가 게임 시작 즉시 노출되며 HP/EXP/레벨/타이머/금화 5개 요소 모두 보인다
- [ ] HP 바 너비가 정확히 280px, 높이 14px이다 (스크린샷 측정)
- [ ] 타이머가 우상단에 흰색 24pt로 표시되고, 10초 이하에서 빨갛게 깜빡인다
- [ ] ⏸ 버튼 또는 ESC 키로 일시정지 메뉴가 뜨며, 게임 진행이 멈춘다
- [ ] 일시정지 메뉴에서 `계속` 누르면 즉시 같은 게임에 복귀한다
- [ ] 일시정지 중 ESC를 다시 누르면 `계속`과 동일하게 동작한다
- [ ] HP가 0이 되면 `쓰러지셨습니다` 화면이 0.25초 페이드인으로 뜬다
- [ ] 5분 생존 시 `5분을 버텨내셨습니다` 화면이 동일 방식으로 뜬다
- [ ] 결과 화면의 4행 통계(생존 시간 / 처치 수 / 최고 레벨 / 획득 금화)가 모두 표시된다
- [ ] `다시 시작` 버튼이 기본 포커스 상태이고, Enter로 즉시 재시작된다
- [ ] `메인 메뉴로` 버튼은 회색이고 눌러도 반응이 없다
- [ ] 결과 화면 표시 중 ⏸ / ESC 입력은 무시된다
- [ ] 모든 한국어 텍스트가 깨지지 않고 존댓말로 통일되어 있다 (모음·받침 정상)
- [ ] 480 × 854 해상도에서 어떤 요소도 화면 밖으로 잘리지 않는다
- [ ] 웹 빌드(HTML5)에서도 위 모든 항목이 동일하게 동작한다
