## 0. 메타 시스템 설계 원칙

| 원칙 | 설명 |
|------|------|
| **런과 메타의 분리** | 런 내 보상(금화·스킬 LV)은 런 종료 시 모두 소실. 메타로 넘어가는 건 **도깨비 구슬 / 신목의 이파리 / 신화 조각 / 도전 진행도**뿐. |
| **누적 완화** | 영구 강화 만렙 시 신규 플레이어 대비 **전투력 상한 +75%**로 캡(과누적 방지). |
| **선택의 무게** | 영구 강화는 리스폨(재분배) 가능하되 **신화 조각 50개** 소모 — 가볍게 굴리지 않는다. |
| **로테이션** | 일일/주간 도전은 **풀(pool)에서 결정적 의사난수 선택** — 서버리스 환경에서도 같은 날짜면 모든 플레이어가 같은 도전을 본다. |
| **막힘 방지** | 신목 단계별 비용 곡선은 "보스 첫 처치 + 메인 진행만으로 자연 도달"을 기본선으로 한다. |
| **데이터 외화** | 모든 수치는 `resources/meta/*.tres`(Godot)에 분리, 코드에 하드코딩 금지. |

## 1. 도깨비 구슬(鬼火珠) 경제

### 1.1 재화 종류 (GDD §7.1 확장)

| 재화 | 영문 키 | 획득 | 용도 | 영구성 |
|------|---------|------|------|--------|
| **도깨비 구슬** | `dokkaebi_orb` | 런 종료 시 정산 | 영구 강화·신목 복구·캐릭터 해금 | ◯ |
| **신목의 이파리** | `shinmok_leaf` | 보스 **최초** 처치 / 신목 단계 보상 | 신목 단계 해금(메타 게이트) | ◯ |
| **신화 조각** | `myth_shard` | 일회성 도전 / 히든 조건 | 도감·히든·강화 트리 리스폨 | ◯ |
| **금화** | `gold` | 런 내 처치/이벤트 | 런 내 상점·이벤트 선택지 | ✕ (런 종료 시 정산 변환) |

### 1.2 획득 경로 — 도깨비 구슬

런 1회 평균 획득량을 **챕터별 + 보너스**로 분해한다. 모든 값은 **베이스값**이며 "행운"·"구슬 보너스" 강화로 가산된다.

#### 1.2.1 런 내 베이스 획득

| 행동 | 베이스 구슬 | 조건 |
|------|-------------|------|
| 일반 몬스터 처치 | 0 | (금화로만 적립, 정산 시 변환) |
| 챕터 1 클리어 | 30 | 보스 처치 포함 |
| 챕터 2 클리어 | 50 | |
| 챕터 3 클리어 | 80 | |
| 챕터 4 클리어 | 120 | |
| 챕터 5 클리어 | 200 | (히든/하드 제외) |
| 히든 챕터 클리어 | 300 | 신목 Lv.4↑ 필요 |
| 하드 챕터 클리어 | (해당 챕터 베이스) × 2 | 신목 Lv.5↑ |
| 무한 모드 5분 생존 | 25 | 신목 Lv.6, 5분 단위 누적 |

#### 1.2.2 보스 처치

| 분류 | 베이스 | 비고 |
|------|--------|------|
| 미니보스 첫 처치 | 50 + 신목 이파리 1 | 같은 미니보스 재처치 시 5 |
| 챕터 보스 첫 처치 | 150 + 신목 이파리 2 | 재처치 시 30 |
| 챕터 5(흑룡) 첫 처치 | 500 + 신목 이파리 5 | 재처치 시 100 |
| 보스 **무피격(노힛)** 보너스 | 추가 +100% | 1회 부활/대시 무적도 허용 안 함 |
| 보스 **시간 제한 처치** | 추가 +50% | 보스별 권장 클리어 시간 이하 |

#### 1.2.3 런 종료 정산 (금화 → 구슬)

```
final_orbs = (chapter_base + boss_bonus)
           + floor(gold_left / 100)            # 100 금화 → 1 구슬
           + achievement_bonus
           + event_bonus
final_orbs *= (1 + orb_multiplier_perm/100)    # 영구 강화 "구슬 보너스"
final_orbs *= (run_difficulty_multiplier)      # 1.0 / 하드 1.5 / 무한 1.2
final_orbs = round(final_orbs)
```

#### 1.2.4 실패(사망) 시

- **베이스 30% 정산**: 도달한 챕터까지의 베이스값 × 0.3, 보스 보너스 0.
- 모은 금화는 × 0.1 변환.
- "끈기" 강화(특수 트리)로 최대 60%까지 회수 가능.

### 1.4 환율 (런 내 ↔ 메타)

| 변환 | 비율 | 비고 |
|------|------|------|
| 금화 → 구슬 (정산) | 100 : 1 | 런 종료 시 |
| 금화 → 구슬 (사망) | 1,000 : 1 | 패널티 |
| 구슬 → 신화 조각 | 200 : 1 | 일일 3회 한정 환전소 (신목 Lv.3↑) |
| 신화 조각 → 구슬 | 1 : 100 | 환전소 (역방향) |
| 이파리 → 구슬 | **불가** | 이파리는 신목 전용 |

### 1.5 인플레이션 방지 규칙

1. **하드 캡(소프트)**: 보유 구슬 > 100,000 시 일반 몬스터 정산 비율 50% 감소(런 클리어 보너스는 정상).
2. **누적 디스카운트**: 같은 보스를 같은 캐릭터로 N번 처치 시 보너스 = `max(10, base / N)`.
3. **하드 모드 강제 사용처**: 신목 Lv.6 도달 후 무한 모드 보상에 **신화 조각**만 지급(구슬 X)하여 후반 인플레 차단.
4. **이벤트 보상 캡**: 보물 상자 1회 최대 300, 신령의 축복 누적 시 50%만 적용.
5. **친밀도/스킨 등 비기능적 사용처**를 후반에 의도적으로 배치해 잉여 구슬을 흡수.
6. **롤백 보호**: 영구 강화 리스폨은 신화 조각으로만 — 구슬로 무한 리스폨/재배치 불가.

## 3. 영구 강화 트리

### 3.1 구조

4개 카테고리 × **각 6~7개 노드** = **총 26개 노드**.
- 트리 형태가 아닌 **카테고리 내 노드 그리드**(선행 조건은 카테고리 내부에서만).
- 각 노드 만렙 = 5레벨 (특수 트리만 3레벨).
- 비용 곡선: `cost(lv) = base * lv^1.6` (반올림), 단 만렙 비용은 §1.3.1 표 참고.

### 3.2 비용 곡선 공식

```
cost(level_target) = round( base * (level_target ^ 1.6) )

예: base=30, LV1→LV2 비용 = round(30 * 2^1.6) ≈ 91
   LV4→LV5 비용 = round(30 * 5^1.6) ≈ 397

누적 비용(만렙 5) ≈ base * (1^1.6 + 2^1.6 + 3^1.6 + 4^1.6 + 5^1.6)
                ≈ base * 26.4
```

각 노드 만렙 누적 = `base * 26.4`. 표에는 `(base, 만렙 누적)`로 표기.

### 3.7 만렙 전투력 상한 (붕괴 방지)

**기본 가정 (신규 플레이어 캐릭터):**
- HP 100, 공격력 10, DPS 1.0, 부활 0, 크리 25% / 크리뎀 150%

**모든 노드 만렙 적용 시:**

| 지표 | 신규 | 만렙 | 배율 |
|------|------|------|------|
| 유효 HP (방어/회피 포함) | 100 | 175 | **×1.75** |
| DPS (크리/공속/시너지 포함) | 1.00 | 1.47 | **×1.47** |
| 부활 | 0 | 3 | +3 |
| 스킬 선택지 | 3 | 6 | ×2 |
| 자원 회수율(런 내) | 100% | 117%(금/경험) | ≈ ×1.17 |

**전투력 종합 상한:**
```
power_index = effective_hp * dps * (1 + 0.2 * 부활)
신규     = 100 * 1.00 * 1.0 = 100
만렙     = 175 * 1.47 * 1.6 = 411.6
배율     = 4.12
```

→ **공식 전투력 상한 = +312%** (신규 대비 약 **4.12배**).
→ 캐릭터 패시브/스킬 시너지를 포함한 **체감 상한 ≈ +75% per stat axis 평균**.
→ **하드캡 가드**: 어떤 단일 스탯도 만렙 강화로 **×2.0을 절대 넘지 않는다** (HP +25%, 공격 +15%, 공속 +10% 등 모두 단일 +50% 이하). 시너지·이벤트로도 단일 스탯 ×2.5를 넘지 않도록 클램프.

**붕괴 방지 추가 규칙:**

1. 만렙 도달은 **신목 Lv.6 이후가 자연스럽도록** 누적 비용 23,000 구슬로 설계.
2. 하드 모드/무한 모드 적 스케일링은 만렙 기준 **체감 난이도 = 신규 일반모드와 같도록 +312% 보정**.
3. 영구 강화 외 다른 영구 버프(이파리, 신화 조각 보너스)는 **없음** — 무제한 누적 차단.
4. 캐릭터별 패시브는 만렙 강화와 **곱연산이 아닌 합연산**으로 적용 (예: HP +25%(강화) + 10%(노을 패시브) → +35%, ×1.375 아닌 ×1.35).

### 4.3 확률 합계 검증

```
시장 22 + 축복 10 + 저주 18 + 상자 15 + 떠돌이 8 + 혈월 5 + 감투 6 = 84%
```
나머지 16%는 **"이벤트 없음"** 상태(자연스러운 공백 시간). 이는 의도된 침묵 구간이며, 시간/조건/이미 발생 여부로 클램프 후 최종 가중치 재정규화.

```python
def roll_event(chapter, time_s, hp_pct, history):
    pool = []
    for ev in EVENTS:
        if not ev.is_allowed(chapter, time_s, hp_pct, history): continue
        pool.append((ev, ev.base_weight * ev.modifier(chapter, hp_pct)))
    total = sum(w for _, w in pool)
    if total == 0 or random() > total/100: return None
    return weighted_choice(pool)
```

## 7. 일일/주간 도전 로테이션 로직

### 7.1 결정성 보장 (서버리스, 결정적 의사난수)

```
seed_day  = hash("daily" + ISO_DATE)            # 예: "daily2026-05-14"
seed_week = hash("weekly" + ISO_WEEK)           # 예: "weekly2026-W20"
```

- ISO 주 기준(월~일).
- `hash`: FNV-1a 32bit 또는 Godot `String.hash()` (구현 일관성만 보장).
- 같은 날짜에는 모든 디바이스가 동일한 도전 목록.

### 7.2 일일 도전 선택 알고리즘

```python
def pick_daily(date_iso, pool=DAILY_POOL_7):
    rng = SeededRng(hash("daily" + date_iso))
    today = rng.sample(pool, 3)                   # 7 중 3, 중복 없음
    return today
```

**규칙**
- 매일 00:00(로컬) 갱신.
- 3개 활성, 각 도전은 24h 안에 1회만 완료 처리.
- 미완료 시 다음 날 자동 폐기(이월 X).
- 일 1회 한정 **즉시 새로고침**: 구슬 100 소비, 한 도전을 풀에서 제거하지 않고 같은 풀에서 다시 추첨(현재 활성 3개와 다른 것 보장).

### 7.3 주간 도전 선택 알고리즘

```python
def pick_weekly(iso_year_week, pool=WEEKLY_POOL_3):
    rng = SeededRng(hash("weekly" + iso_year_week))
    # 풀 3개 중 1개 활성, 단 4주 연속 같은 도전 금지
    last4 = load_last4_weeks()
    candidates = [w for w in pool if w not in last4] or pool
    return rng.choice(candidates)
```

**규칙**
- 매주 월요일 00:00 갱신.
- 1개 활성, 일주일간 동일 도전 유지.
- 동일 도전이 4주 내 다시 등장 금지(편향 방지).
- 완료 후 즉시 보상 지급, 동일 주 재완료는 점수만 표시.

### 7.4 진행도 저장 구조

### 7.5 UI/UX

- 메인 메뉴 좌측 패널 "도전" 탭.
- 카드 형태: 아이콘 / 제목 / 진행도 바 / 보상 / 만료까지 남은 시간.
- 완료 시 카드 황금 테두리 + "수령" 버튼. 미수령은 다음 갱신 시점까지 가능, 갱신 이후 자동 회수(이월 없음).
