## 0. 본 문서의 사용 규칙

### 0.1 수치 표기 규약
- `INT` = 지능, `ATK` = 공격력, `LV` = 스킬 레벨(1~5).
- 데미지 표기 예: `20 + INT*0.5` (값은 1회 틱/타격 기준).
- 지속 대미지(DoT)는 **초당** 값으로 표기: `5/s × 4s` = 4초간 초당 5.
- 거리/범위는 모두 **픽셀** 단위. 화면 전체 = 1280×720.
- 쿨다운(CD)은 **초** 단위. `LV1 CD 8s → LV5 CD 5s` 형식.
- 확률은 % 단위. 1.0 = 100%.

### 0.2 등급별 출현 확률 (레벨업 3택 풀)
| 등급 | 기본 등장 확률 | "행운" 보정 후 최대 |
|------|----------------|---------------------|
| 일반 (Common) | 60% | 40% |
| 고급 (Rare) | 30% | 40% |
| 전설 (Legendary) | 10% | 20% |

- 행운 1포인트당 고급 이상 확률 +2%p (GDD §7.3 영구강화 "행운" 표 준수).
- 전설은 **챕터 3 클리어 후**부터 풀에 진입(메타 진행 잠금).

### 0.3 레벨업 3택 시스템 정합성 규칙

본 문서가 정의하는 30종은 **레벨업 3택**(GDD §3.3)에서 다음 규칙에 따라 후보로 산출된다:

| 규칙 | 내용 |
|------|------|
| R1 | 한 화면(3택)에 같은 스킬은 절대 중복 등장 금지 |
| R2 | 이미 보유 + Max 레벨인 스킬은 후보에서 제외 |
| R3 | 보유 중인 스킬 5종 이상이면 그 중 미만렙 1종이 반드시 1슬롯에 등장 (성장 보장) |
| R4 | 시너지 진화 가능 조건이 충족되면 **진화 카드**가 일반 3택을 대체하여 등장 (확률 35%) |
| R5 | 저주 선택지는 5분당 1회 최대, 진화 카드와 동시 등장 금지 |
| R6 | 같은 속성 스킬을 4개 이상 보유한 경우, 다른 속성 스킬이 3택 중 최소 1슬롯에 등장 (편중 완화) |
| R7 | 보유 스킬 수 한도 = 8개 (액티브 6 + 패시브 2). 한도 도달 후엔 기존 보유 스킬의 레벨업 카드만 등장 |

전설 스킬은 **한 런당 최대 2종**까지 획득 가능(전설 인플레이션 방지). 시너지 진화 결과물은 이 카운트에 포함되지 않는다.

### 0.4 발동 방식 분류
- **자동(Auto)**: 쿨다운 도래 시 알아서 발동. 대부분의 스킬.
- **수동(Active)**: 화면 탭(모바일) / 스페이스(웹). 별도 슬롯 1개에 한정. 강력하지만 입력 필요.
- **패시브(Passive)**: 항상 적용되는 스탯/조건 효과. 직접 데미지 없음.

### 0.5 상생/상극 보정 (GDD §5.2 준수)
- 상생 페어 동시 장착 시 두 스킬 모두 **최종 효과 +25%** (데미지/힐/지속시간 포함).
- 상극 페어 동시 장착 시 두 스킬 모두 **최종 효과 -10%**, 단 **시너지 진화 카드 등장 확률 +50%**.
- 본 문서의 모든 수치는 **상생/상극 보정 전(중립)** 기준.

#### 진화 1. 산불 (화 + 목)

- **속성 분류**: 화/목 융합 (상생)
- **통합 효과**: 신목이 발화하면서 화면 1/3 영역에 "불타는 숲"을 생성. 입장한 적은 지속 화상 + 덩굴에 속박(둔화 70%).

- **쿨다운**: 단계1 25s / 단계2 20s / 단계3 16s
- **범위/사거리**: 화면 좌하단 또는 우하단 1/3 (약 640×360px), 캐릭터 위치에 따라 결정
- **시각 이펙트**: 푸른 잎과 붉은 불꽃이 픽셀로 뒤섞임, 적 위 작은 새 픽셀(놀라 도망)
- **연출/메시지**: "신목이 분노로 타오릅니다." + 한자 "山火"

## 8. 레벨업 3택 시스템과의 정합성 정리

### 8.1 후보 풀 산출 알고리즘 (의사 코드)

```
function getLevelUpChoices(player):
    pool = all_skills.filter(unlocked_by_chapter)

    # R2: 만렙 제외
    pool = pool.exclude(player.skills.filter(level == max))

    # R7: 슬롯 한도 도달 시
    if len(player.skills) >= 8:
        pool = player.skills.filter(level < max)

    # R4: 진화 가능 페어 검사
    evolutionCandidates = checkEvolutionPairs(player.skills)
    if evolutionCandidates and roll(0.35 * (상극이면 1.5 else 1.0)):
        return chooseOneEvolutionCard(evolutionCandidates) + drawNormal(2)

    # R3: 미만렙 보장
    choices = []
    if len(player.skills) >= 5:
        ownUnderLevel = player.skills.filter(level < max)
        if ownUnderLevel:
            choices.append(weightedRandom(ownUnderLevel))

    # R6: 속성 편중 완화
    dominantElement = player.skills.groupBy(element).maxCount()
    if dominantElement.count >= 4:
        otherPool = pool.filter(element != dominantElement)
        choices.append(weightedRandom(otherPool))

    # 남은 슬롯 채우기 (R1 중복 금지)
    while len(choices) < 3:
        candidate = weightedRandom(pool, byGradeWeights(player.luck))
        if candidate not in choices:
            choices.append(candidate)

    # R5: 저주 카드 (5분당 1회, 진화 없을 때만)
    if not hasEvolutionCard(choices) and timeSinceLastCurse > 300:
        replaceWith(choices, weightedRandom(curseDeck), prob=0.2)

    return choices
```
