> 작성: PM Agent

## 0. Executive Summary

- **모델**: F2P + Hybrid (보상형 광고 + IAP + 시즌 배틀패스)
- **핵심 원칙**: **페이투윈 0%** — 결제는 시간 단축, 편의성, 코스메틱으로만 이어진다.
- **광고 정책**: 보상형 중심, 배너 없음, 인터스티셜은 최소화.
- **IAP 가격대**: 한국 표준 티어(1,100 / 3,300 / 5,500 / 9,900 / 33,000 / 55,000) 기반.
- **시즌**: **6주 1시즌**, 무료/유료 트랙 40레벨, 유료 9,900원.
- **핵심 KPI (출시 6개월 목표)**:
  - D7 잔존 **25%**, D30 잔존 **12%**
  - ARPDAU **₩200** (광고 40% + IAP 60%)
  - 보상형 광고 시청률 **60%** (DAU 중)
  - 결제 전환율(D30) **5%**
- **윤리 가이드라인**: 미성년자 보호, 다크패턴 금지 7종, 가챠 확률·천장 공개, 환불 SLA 5일.
- **운영 케이던스**: 시즌 6주 / 격주 이벤트 / 월간 마이너 패치 / 분기 신규 모드 / 연 1회 메이저.

## 1. 수익 모델 유형 결정

### 1.1 후보 비교

| 모델 | 장점 | 단점 | 깨비런 적합도 |
|------|------|------|---------------|
| **F2P + 광고만** | 진입장벽 0, 캐주얼 사용자 친화 | 한국 시장 ARPDAU 낮음(₩50–₩100), LTV 한계, 광고 SDK 의존 | △ (보조 수익원으로만) |
| **F2P + IAP만** | 결제 전환 사용자 LTV ↑, UX 깔끔 | 진입장벽 ↑, 페이투윈 인식 시 이탈 | △ (배틀패스 부족) |
| **Paid (유료 판매)** | 진입 시점 매출 확정, UX 가장 깔끔 | 한국 모바일 시장에서 1% 미만 점유, UA 매우 어려움 | ✗ |
| **Hybrid (광고 + IAP + 패스)** | 광고가 진입장벽 낮추고, IAP가 LTV 끌어올림, 패스가 잔존 유지 | 시스템 복잡도 ↑, 운영 부담 | ✅ **채택** |

### 1.2 최종 선택: **F2P + Hybrid**

**근거**

1. **시장 표준** — 한국 모바일 게임 매출 상위 100개 중 약 82%가 Hybrid 모델 (2025년 KOCCA 보고서 기준).
2. **장르 적합성** — 깨비런은 캐주얼 로그라이크. 1회 플레이 8–15분의 짧은 세션 → 보상형 광고가 자연스럽게 녹는다. 메타 진행 시스템(영구 강화 트리, 신목 복구)이 길어 IAP/패스가 진가를 발휘한다.
3. **사용자 세그먼트 분산** — 무과금(광고만), 소과금(스타터/구슬 묶음), 충성 유저(배틀패스), 헤비 스펜더(프리미엄 패스)까지 세그먼트별 수익화가 가능하다.
4. **PvP 부재** — 깨비런은 PvP가 없는 PvE 로그라이크. **페이투윈 회피가 구조적으로 쉽다** — 결제가 다른 사용자에게 영향을 주지 않는다.

### 1.3 모델 채택 시 핵심 제약

- 영구 강화 트리(`meta-systems-spec.md` §3)의 단일 스탯 ×2 / 전체 전투력 ×4.12 하드캡은 **절대 결제로 우회 불가**.
- 부활/회생은 **광고 시청으로만** 제공, IAP 부활 상품 **금지**.
- 코어 게임플레이 진행에 필요한 자원(구슬, 이파리)은 결제 없이도 정상 시간 안에 획득 가능해야 한다.

## 2. 광고 시스템

### 2.1 광고 유형 정책

| 유형 | 사용 여부 | 비고 |
|------|-----------|------|
| 보상형 비디오 (Rewarded Video) | ✅ **메인** | 사용자가 보상을 위해 자발적으로 시청 |
| 인터스티셜 (Interstitial) | ✅ 보조 | 빈도 강하게 제한 |
| 배너 (Banner) | ❌ **사용 안 함** | 화면 점유 + 캐주얼 게임 UX 저해 |
| 네이티브 광고 (Native) | ⚠ 제한적 사용 | 메뉴 화면 1개 슬롯만, 옵션으로 끌 수 있음 |
| 앱오픈 (App Open) | ❌ 사용 안 함 | 첫인상 저해, 잔존율 ↓ |

### 2.2 보상형 광고 배치 (5종)

#### B-1. 사망 후 부활
- **배치**: 챕터 보스전 / 일반 챕터에서 사망 시 1회 부활 가능
- **보상**: 동일 위치에서 HP 50% / 게이지 50%로 부활
- **빈도 제한**: **1런(run)당 1회**, 추가 부활 불가 (페이투윈 회피)
- **UX**: "광고 보고 부활" / "포기" 2버튼, 5초 카운트다운

#### B-2. 챕터 클리어 보상 더블링
- **배치**: 챕터 클리어 직후 보상 정산 화면
- **보상**: 획득 구슬 ×2 (이파리는 더블링 X — 희소성 유지)
- **빈도 제한**: **챕터당 1회**, 일일 캡 없음

#### B-3. 메뉴 추가 구슬 (Daily Reward)
- **배치**: 메인 메뉴 "도깨비 구슬 시청" 슬롯
- **보상**: 500 구슬 / 회 (구슬 가치 상 약 5분 플레이 분량)
- **빈도 제한**: **일일 5회**, 각 시청 간 최소 5분 간격

#### B-4. 이파리 가챠
- **배치**: 메인 메뉴 "신목의 이파리" 슬롯
- **보상**: 일일 1회 무료 시청 시 이파리 1장 (확률: 70% 일반잎 / 25% 은잎 / 5% 금잎)
- **빈도 제한**: **일일 1회**, 무료 가챠 (IAP 가챠 없음)

#### B-5. 영구 강화 노드 즉시 적용 가속
- **배치**: 영구 강화 트리에서 노드 업그레이드 직후
- **보상**: 다음 런에서 해당 노드 효과 +10% 일시 강화 (1회만)
- **빈도 제한**: **노드 업그레이드당 1회**
- **주의**: 영구 강화량 자체를 늘리지 않음 → 페이투윈 회피

### 2.3 인터스티셜 배치 (2종)

#### I-1. 챕터 전환 인터스티셜
- **배치**: 챕터 ↔ 챕터 사이 진입 로딩 시
- **빈도**: **챕터 3회당 1회** (사용자 피로도 관리)
- **소프트 온보딩**: 신규 사용자 첫 30분간 노출 0회

#### I-2. 메뉴 ↔ 게임 전환 인터스티셜
- **배치**: 메뉴 → 게임 시작 직전
- **빈도**: **일일 최대 3회**, 두 노출 간 최소 90초 간격

### 2.4 빈도 제한 종합

| 광고 유형 | 일일 캡 | 최소 간격 | 비고 |
|-----------|---------|-----------|------|
| 보상형 (전체) | 일일 15회 | 30초 | B-1~B-5 합산 |
| 인터스티셜 (전체) | 일일 5회 | 90초 | I-1+I-2 합산 |
| 신규 사용자 (D1) | 인터스티셜 0회 | — | 소프트 온보딩 |
| 광고 제거 결제자 | 0회 (광고형 보상만 유지) | — | §3.2 참조 |

### 2.5 광고 SDK / 미디에이션

- **권장**: AppLovin MAX (한국 eCPM 강세) + AdMob Mediation
- **백업**: Unity Ads (게임 광고 친화, 보상형 안정)
- **결정 권한**: Solution Architect (이 문서는 권고안)
- **기대 eCPM** (한국, 2025년 기준):
  - 보상형: ₩4,000–₩5,500
  - 인터스티셜: ₩2,500–₩4,000
  - 네이티브: ₩1,500–₩2,500

### 2.6 UX 저해 최소화 방침

1. **자발성 우선** — 광고 시청은 항상 선택. 강제 노출 X (인터스티셜 제외).
2. **닫기 버튼 정책** — 광고 시작 후 3초 뒤 X 버튼 활성. 미스클릭 유도 다크패턴 금지.
3. **오프라인 폴백** — 광고 로드 실패 시 동일 보상 100% 지급 (사용자 비난 회피).
4. **광고 컨텐츠 필터** — 19금/도박/유흥/경쟁사 광고 차단 (SDK 옵션).
5. **광고 제거 상품 제공** — 9,900원 결제 시 인터스티셜/네이티브 영구 제거. 단, 보상형은 유지 (사용자 자율).

## 3. IAP 시스템

### 3.1 설계 원칙

1. **페이투윈 회피**
   - 영구 강화 트리 노드는 구슬로만 해금 (구슬 IAP 가능하지만 §4.4 하드캡 적용).
   - 시즌 패스 보상 비율: **코스메틱 70% / 게임플레이 30%**.
   - 한정 시즌 보상은 다음 시즌에 일반 보상으로 재등장 (FOMO 완화).
2. **결제 옵션 단순화** — 동시 진열 패키지 7–9종 이내, 정보 과부하 회피.
3. **결제 후 가치 즉시 체감** — 모든 패키지는 결제 직후 사용 가능. 출금 지연 X.
4. **첫 결제 친화** — 스타터 팩(3,300원)을 강력하게 노출하여 첫 결제 장벽을 낮춤.

### 3.3 가격 책정 근거

- **App Store / Google Play 한국 가격 티어** 활용 (1,100 / 2,200 / 3,300 / 5,500 / 9,900 / 11,000 / 22,000 / 33,000 / 55,000).
- **캐주얼 게임 ARPPU 평균 (2025 한국)**: 월 8,000–12,000원 → 배틀패스 9,900원이 자연스러운 앵커 가격.
- **소액 결제 비중**: 1,100원 + 3,300원 묶음이 전체 결제 건수의 60%+ 차지 예상.

## 4. 시즌제 / 배틀패스 운영

### 4.1 시즌 구조

| 항목 | 정책 |

## 9. 리스크 & 대응

### 9.1 광고 의존도 리스크

- **리스크**: 사용자가 "구슬 IAP = 페이투윈" 인식 → 무과금 사용자 이탈
- **대응 1**: 구슬 사용처를 영구 강화뿐 아니라 코스메틱/이벤트로 분산

- **대응 3**: 명시적 안내 — "캐릭터 강화는 실력으로도 도달 가능"
- **대응 4**: PvP 부재 → 결제가 다른 사용자에게 영향 X (구조적 보호)

### 9.3 시즌 콘텐츠 부담

- **리스크**: 6주 시즌 케이던스를 지속하기 어려움 (특히 1년차 후반)

- **대응 2**: 시즌 4부터 실시간 제작 시 외주 활용 (코스메틱 스킨)
