# KKAEBI RUN — 아트 에셋 생성 사양 (Single Source of Truth)

> **버전**: v1.0
> **상태**: 확정 (PD 핸드오프)
> **목적**: 모든 비주얼 에셋 자동 생성 스크립트(스프라이트 합성기, 팔레트 변환기, 9-slice 패커 등)는 본 문서를 단일 소스로 참조한다.
> **자매 파일**: [`art-palette.json`](./art-palette.json) — 머신 리더블 팔레트 정의. 본 문서와 항목이 충돌하면 본 문서가 우선이며, JSON을 갱신한다.
> **관계**: 본 문서는 `art-style-guide.md`(상위 비전 SSOT)의 *생성 측면* 구현 사양이다. 아트 비전 충돌 시 `art-style-guide.md`를 정본으로 두고 본 문서를 갱신한다.

## 1. 글로벌 사양

### 1.1 스타일 룰

| 항목 | 값 | 비고 |
|------|----|----|
| 비례 | 1:1 ~ 1:1.5 (head:body) | SD 비례 — 챕터/캐릭터 무관 고정 |
| 실루엣 | 둥근 형태 | 뾰족·세장형은 보스/특정 종에 한정 |
| 외곽선 | `#1A1A1A`, 1px | 모든 캐릭터/몬스터/보스에 필수 |
| 디더링 | **사용 안 함** | 색 전이는 단일 픽셀 경계로 처리 |
| 종이 배경 | `#F4E8D0` | 미리보기/픽업/UI 백드롭 기본값 |

### 1.2 베이스 팔레트 (8색 고정)

| 토큰 | HEX | 용도 |
|------|-----|----|
| `hanji_yellow` | `#D4A574` | 한지·UI 패널 채움·토(土) 액센트 |
| `jade`         | `#5BAD8A` | 신령·식물·치유 |
| `ink`          | `#2D2D2D` | 그림자·먹선·텍스트 강조 |
| `vermillion`   | `#E85D3A` | 위험·화(火)·HP·로고 |
| `sky`          | `#7BC8F6` | 수(水)·바람·UI 액센트 |
| `gold`         | `#F0C75E` | 신성·보상·골드 |
| `ink_outline`  | `#1A1A1A` | 1px 외곽선 전용 |
| `paper_bg`     | `#F4E8D0` | 종이 배경·UI 베이스 |

> 캐릭터/몬스터/보스/이펙트의 모든 메인·서브·포인트 색은 위 8색을 기본으로 하며, 본 문서가 명시한 보조 HEX(예: `#8B5A2B` 같은 캐릭터 전용 색)만 허용한다. 자유 색상 추가는 금지.

### 1.3 해상도

| 분류 | 해상도(px) |
|------|-----------|
| 캐릭터(플레이어 6종) | **32×32** |
| 일반 몬스터(M01~M53) | **32×32** |
| 보스/미니보스(b01~b06, mb01~mb05) | **64×64** |
| 배경 타일 | **16×16** |
| 스킬/타격 이펙트 | **32×32** |
| UI 9-slice (소) | **64×64** |
| UI 9-slice (와이드) | **128×64** |

### 1.4 스프라이트시트 레이아웃 (가로 strip 고정)

모든 스프라이트시트는 좌→우 방향의 단일 가로 strip이다. 행은 1행 고정.

| 대상 | 프레임 구성 | 총 프레임 | 시트 크기(px) |
|------|------------|----------|---------------|
| 캐릭터 | idle(4) + run(6) + attack(4) + hit(2) + die(4) | **20** | **640×32** |
| 일반 몬스터 | idle(4) + attack(3) + die(3) | **10** | **320×32** |
| 보스/미니보스 | idle(4) + attack(4) + die(4) | **12** | **768×64** |
| 스킬 이펙트 | 단일 시퀀스 | **4** | **128×32** |

프레임 순서는 위 표 그대로(좌측이 idle 첫 프레임). 인덱스 산출은 0-base.

**생성 노트**:
- 헬멧/모자/머리띠 같은 식별 요소는 idle의 모든 프레임에서 동일한 픽셀 위치를 유지한다(±1px 이내).
- run 6프레임은 좌·우 발 교차 패턴: f0/f3 = 양발 접지, f1·f4 = 들기, f2·f5 = 최고점.
- attack 4프레임은 anticipation(1) → strike(1) → impact(1) → recover(1) 순.

## 7. 생성 스크립트 인터페이스 (참조 규약)

본 문서를 참조하는 생성 스크립트는 다음을 보장한다:

1. **단일 소스**: 팔레트 토큰은 `art-palette.json` 의 키를 그대로 사용한다(예: `palette.base.vermillion`). 하드코딩된 HEX 금지.
2. **시트 검증**: 출력 PNG의 (width × height)가 §1.4 표와 정확히 일치한다.
3. **외곽선 검증**: 알파 0 픽셀에 인접한 알파 >0 픽셀의 RGB는 반드시 `#1A1A1A`.
4. **디더링 금지**: 인접 두 픽셀의 RGB 차가 임계값(80) 미만인 경계가 2픽셀 폭 이상 반복되면 디더 의심 → 빌드 실패.
5. **카탈로그 정합**: 몬스터/보스 ID 누락 검사 — `.tres`에는 있는데 본 표에 없으면 빌드 경고.

검증 도구는 `scripts/art_validate.py`(미작성, 후속 태스크) 에서 본 문서를 파싱하여 적용한다.
