### 1.3 핵심 진단

> **현재는 "심장은 뛰는데 얼굴이 없는" 상태다.**
> 모든 시스템 데이터(HP, EXP, 레벨, 잔여 시간, 사망)는 흐르고 있지만 플레이어는 그 어떤 것도 화면에서 확인할 수 없다. 죽었는지조차 모른 채 게임이 멈춘다.

이 상태에서 가장 큰 가치를 만들어내는 변화는 **(1) 보이게 만들기 + (2) 한 판을 끝낼 수 있게 만들기**다. 그 다음에야 "스킬을 고르는 재미"(레벨업 3택)가 의미를 가진다.

## 2. 다음 단계 스코프 결정

### 2.1 결정 — 이번 다음 단계에서 구현할 범위 (우선순위 순)

> 범위 폭주를 막기 위해 **2개**만 진행한다. 스킬 3택은 다음 다음 단계로 미룬다.

#### 🥇 P1. 인게임 HUD
**왜 1순위인가**
- 모든 signal(`hp_changed`, `exp_changed`, `level_up`)이 이미 emit되고 있음 — 받기만 하면 됨
- 구현 비용 최소(노드 배치 + signal 연결), 플레이어 체감 가치 최대
- HUD 없이는 P2(결과 화면)에서 보여줄 수치도 신뢰할 수 없음 (검증 불가)

#### 🥈 P2. 사망 / 스테이지 종료 처리 + 결과 화면 + 재시작
**왜 2순위인가**
- 현재 `Player.died`만 emit되고 그 뒤가 아예 없음 → 새로고침 강제
- 5분 생존(`time_left <= 0`) 처리도 없음 → 한 판이 "끝"날 수 없음
- 이게 있어야 비로소 짧은 한 판 → 다시 한 판의 루프가 성립함

#### ⛔ 이번에 하지 않을 것 (의도적 미포함)
| 항목 | 이번에 안 하는 이유 |
|------|--------------------|
| 스킬 3택 UI + 5속성 스킬 | 스킬 5개를 디자인+구현+밸런싱하면 단독 스프린트 분량. HUD/사망처리 위에 얹는 것이 안전 |
| 일시정지 메뉴 | 결과 화면이 들어오면 사실상 강제 일시정지 효과를 얻음. 정식 일시정지는 스킬 단계에서 함께 |
| 메인 메뉴 | 결과 화면에서 "다시 시작"만 있으면 MVP 검증은 가능. 메인 메뉴는 캐릭터 선택과 묶어 Phase 2 |
| 사운드 플레이스홀더 | 비주얼 피드백 확립 뒤에 시도. 지금은 데드 코드 증가 위험 |
| 금화 시스템 | Phase 2 (GDD §7.1, §13.2) |
| 입력맵 보강 | 일시정지가 빠지므로 `pause` 액션은 다음 단계에서 함께 등록 |

### 2.2 비범위 항목의 회피 처리

- `Main._process`에서 `time_left <= 0`일 때 **즉시 "스테이지 클리어" 상태로 전환**한다 (3.2 참조). 빈 종료가 되지 않도록 한다.
- `Player.died`는 **결과 화면 띄우는 데에만** 사용한다. 메타 정산·금화 합산 등은 호출하지 않는다.

## 3. 항목별 상세 스펙

### 3.1 P1 — 인게임 HUD

#### 파일 / 노드 구조
```
scenes/ui/hud.tscn          # CanvasLayer 루트, 항상 카메라 위
scripts/ui/hud.gd
```
`Main` 씬에 `HUD`(CanvasLayer) 자식으로 인스턴스. `Main._ready()`에서 `$Player`의 signal을 `HUD` 메서드에 연결한다.

#### 표시 요소 및 위치 (해상도 480 × 854 세로)

| 요소 | 위치 (anchor) | 크기 | 표시 내용 |
|------|--------------|------|----------|
| HP 바 | 좌상단 — `(16, 16)` | `200 × 14` | 빨강 채움. 우측 끝에 `현재/최대` 텍스트(예: `78 / 100`) |
| 레벨 표시 | HP 바 바로 아래 — `(16, 36)` | auto | `레벨 3` (한국어, 존댓말 톤은 아니지만 시스템 라벨이라 평서체) |
| 타이머 | 상단 중앙 — anchor top, x center | auto | `남은 시간 04:37` 형식, `mm:ss` |
| EXP 바 | 화면 하단 가로 풀폭 — bottom, y=`-12` | `(viewport_w - 32) × 6` | 노랑 채움. EXP만 표시 (수치 텍스트는 없음 — 시인성 우선) |

> **금화 / 스킬 아이콘 슬롯은 Phase 1에서 표시하지 않는다.** GDD §9.3 HUD 정의는 따르되, 미구현 시스템(금화·스킬)을 위한 빈 슬롯도 두지 않는다 — 사용자에게 "비어 있다"는 인상을 주지 않기 위함.

#### 색상 (GDD §9.1 오방색 팔레트 준수)
- HP 바: 채움 `#E03C3C`, 배경 `#3A1A1A`
- EXP 바: 채움 `#E0C23C`, 배경 `#3A331A`
- 텍스트: `#F5F1E6` (한지 톤 흰색)

#### 사용자 노출 한국어 문구
| 위치 | 문구 | 비고 |
|------|------|------|
| 타이머 라벨 | `남은 시간` | 라벨이라 평서 명사형 |
| 레벨 라벨 | `레벨 {N}` | `N`은 정수, 1부터 시작 |
| HP 표기 | `{cur} / {max}` | 숫자만, 라벨 없음 |

#### 동작
- `hp_changed(hp, max_hp)` → HP 바 채움 비율 = `hp/max_hp`, 텍스트 갱신
- `exp_changed(exp, exp_to_next)` → EXP 바 채움 비율 = `exp/exp_to_next`
- `level_up(level)` → 레벨 텍스트 갱신, `Tween`으로 0.2초간 1.0 → 1.2 → 1.0 스케일 연출 (GDD §9.2 "레벨업 — 화면 번쩍 + 도깨비 방망이 아이콘 회전"의 축소판)
- `Main`의 `time_left` → HUD가 매 프레임 polling. `00:00`이 되기 전 마지막 10초는 텍스트 색을 `#E03C3C`로 변경

#### 수용 기준
1. 게임 시작 직후 HP 100/100, EXP 0/10, 레벨 1, 남은 시간 05:00 이 모두 표시된다.
2. 적과 접촉해 피해를 입으면 HP 바와 숫자가 즉시 줄어든다.
3. 경험치 보석을 먹으면 EXP 바가 채워지고, 가득 차면 레벨 표시가 1 증가하며 살짝 커졌다 줄어드는 연출이 보인다.
4. 시간이 줄어들고, 04:50 → 04:49 처럼 매초 갱신되며 마지막 10초는 빨갛게 표시된다.

### 3.2 P2 — 사망 / 스테이지 종료 + 결과 화면 + 재시작

#### 파일 / 노드 구조
```
scenes/ui/result_screen.tscn    # CanvasLayer, 전체 화면 오버레이
scripts/ui/result_screen.gd
```

#### 트리거 조건
| 조건 | 결과 화면 모드 | 처리 위치 |
|------|--------------|----------|
| `Player.died` emit | **사망** 모드 | `Main`이 signal 수신 |
| `Main.time_left <= 0.0` 진입 | **클리어** 모드 | `Main._process` 안에서 명시적으로 전환 |

#### 공통 동작
- 결과 화면이 떠 있는 동안 **게임은 일시정지** — `get_tree().paused = true`
- `Main`, `Player`, `Enemy*`, `ExpGem` 모두 `process_mode = PROCESS_MODE_PAUSABLE` (기본값) 유지
- 결과 화면 본인은 `PROCESS_MODE_ALWAYS`로 설정 (입력 받아야 함)

#### 사망 모드 — 화면 구성

| 요소 | 문구 / 표시 |
|------|------------|
| 타이틀 | `쓰러지셨습니다` |
| 부제 (있다면) | `다음에는 더 멀리 가실 수 있습니다` |
| 통계 1 | `생존 시간   {경과시간 mm:ss}` |
| 통계 2 | `도달 레벨   레벨 {N}` |
| 통계 3 | `처치 수     {kill_count}` |
| 버튼 1 (기본 포커스) | `다시 시작` |
| 버튼 2 | `메인 메뉴로` |

> **`메인 메뉴로` 버튼은 이번 단계에서 비활성화(disabled) + 라벨 회색** 처리한다. 메인 메뉴 씬이 아직 없으므로 클릭해도 동작하지 않으면 혼란을 준다. 자리는 잡아두되 누를 수 없게 한다.

#### 클리어 모드 — 화면 구성

| 요소 | 문구 / 표시 |
|------|------------|
| 타이틀 | `5분을 버텨내셨습니다` |
| 부제 | `깨비런 첫 스테이지를 클리어하셨어요` |
| 통계 1 | `처치 수     {kill_count}` |
| 통계 2 | `최종 레벨   레벨 {N}` |
| 버튼 1 (기본 포커스) | `다시 시작` |
| 버튼 2 | `메인 메뉴로` (이번엔 비활성화) |

#### `kill_count` 집계 방법
- `Enemy.die()` 안에서 `EventBus`-스타일 싱글톤 없이 단순화: `get_tree().current_scene` 가 `Main`이라는 전제 하에 `Main`에 `kill_count: int` 필드를 두고, `Enemy.die()`가 `current_scene.kill_count += 1` 시도.
- 안전을 위해 `if main and "kill_count" in main: main.kill_count += 1` 가드.
- Phase 1 한정의 임시 합산이므로 별도 시스템화는 하지 않는다.

#### 사용자 노출 한국어 문구 (전체 목록)

> **모든 문구는 존댓말로 통일.** (조직 메모리 `feedback_tone.md` 준수)

| 키 | 한국어 |
|----|--------|
| `result.died.title` | `쓰러지셨습니다` |
| `result.died.subtitle` | `다음에는 더 멀리 가실 수 있습니다` |
| `result.clear.title` | `5분을 버텨내셨습니다` |
| `result.clear.subtitle` | `깨비런 첫 스테이지를 클리어하셨어요` |
| `result.stat.survived` | `생존 시간` |
| `result.stat.level_reached` | `도달 레벨` |
| `result.stat.level_final` | `최종 레벨` |
| `result.stat.kills` | `처치 수` |
| `result.level_format` | `레벨 {N}` |
| `result.button.restart` | `다시 시작` |
| `result.button.main_menu` | `메인 메뉴로` |

> 문구는 `result_screen.gd` 상단 `const` 블록에 모아둔다. i18n 시스템은 Phase 1 범위 밖이므로 상수 그대로 사용.

#### `다시 시작` 동작
1. `get_tree().paused = false`
2. `get_tree().reload_current_scene()`

`Main` 씬을 통째로 다시 로드하므로 상태 초기화는 자동. 별도 reset 메서드 작성 불필요.

#### `메인 메뉴로` 동작 (이번 단계)
- 비활성화. 클릭 불가. 다음 단계에서 활성화 예정.

#### 키보드/입력
- 결과 화면 표시 후 첫 프레임에 `다시 시작` 버튼에 `grab_focus()` 호출.
- 엔터/스페이스로 다시 시작 가능해야 함 (Godot Button 기본 동작).

#### 시각 처리
- 결과 화면 떴을 때 게임 화면 위에 `Color(0, 0, 0, 0.6)` 어둠 패널을 깐 뒤 그 위에 패널/버튼.
- 패널 배경: `#1A1612` (한지 어두운 톤), 테두리 1px `#E0C23C`.
- 트윈으로 0.25초 페이드인.

#### 수용 기준
1. 적과 부딪혀 HP가 0이 되면 게임이 멈추고 `쓰러지셨습니다` 화면이 나타난다.
2. 화면에 생존 시간(`mm:ss`), 도달 레벨, 처치 수가 표시된다.
3. `다시 시작` 버튼을 클릭하거나 엔터를 누르면 게임이 처음부터 다시 시작된다.
4. 5분을 끝까지 버티면 `5분을 버텨내셨습니다` 화면이 나타나고, 동일하게 다시 시작이 가능하다.
5. `메인 메뉴로` 버튼은 회색으로 보이고 눌리지 않는다.
6. 결과 화면이 떠 있는 동안 적은 움직이지 않고, 타이머도 멈춰 있다.
